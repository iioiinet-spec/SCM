2. 데이터 압축

손실 압축 vs 무손실 압축

학습지 1) 손실압축과 무손실 압축의 특징

Q 손실압축과 무손실 압축의 특징을 비교 설명하시오

🔻 손실 압축 (Lossy)

  • 압축 과정에서 데이터 일부를 버려서 복원 시 원본과 완전히 같지 않음
  • 대신 압축률이 매우 높음 (용량을 크게 줄일 수 있음)
  • 사람이 잘 인식하지 못하는 정보(고주파 음역, 미세한 색상차 등)를 제거
  • 예: JPEG(이미지), MP3(음악), MPEG(동영상)

🔺 무손실 압축 (Lossless)

  • 압축해도 데이터 손실이 전혀 없어 원본을 완벽히 복원 가능
  • 압축률은 손실 압축보다 낮음
  • 반복되는 패턴·중복 데이터를 효율적으로 표현하는 방식 사용
  • 예: ZIP, PNG, 런렝스 압축, 허프만 코딩

💡 핵심 비교 포인트: 복원 가능 여부와 압축률은 서로 트레이드오프 관계입니다. 시험에서는 "문서·실행파일처럼 정확해야 하는 데이터엔 무손실, 이미지·음악처럼 약간의 손실이 허용되는 데이터엔 손실 압축을 사용한다"는 활용 맥락도 함께 출제될 수 있어요.

2. 데이터 압축

런렝스(Run-Length) 압축 문제풀이

학습지 2) 다음 데이터를 런렝스 압축하시오

Q1 HELLO 를 런렝스 압축하면?

원본: H E L L O → 압축 전 byte 수, 압축 후 byte 수를 모두 구하세요.

문자HELO
반복횟수1121

➡️ 압축 결과: H1E1L2O1

압축 전: HELLO = 5 byte (문자 1개 = 1byte 가정)

압축 후: H1E1L2O1 = 8 byte (문자+숫자 쌍 4개 × 2byte)

⚠️ 주의: HELLO처럼 같은 문자가 적게 반복되면, 런렝스 압축이 오히려 원본보다 용량이 커질 수 있습니다. 이게 이 문제의 핵심 포인트예요 — 런렝스 압축은 "같은 값이 길게 연속될 때"만 효과적입니다.

Q2 GOOOOGLE 를 런렝스 압축하면?

원본: G O O O O G L E (O가 4번 연속)

문자GOGLE
반복횟수14111

➡️ 압축 결과: G1O4G1L1E1

압축 전: GOOOOGLE = 8 byte

압축 후: G1O4G1L1E1 = 10 byte (5쌍 × 2byte)

이 경우도 글자 종류가 많아(G가 두 번 끊어져 등장) 압축 효율이 낮습니다. 만약 GGGGOOOO처럼 큰 덩어리로 반복되면 압축 효과가 커집니다. 시험에서 "런렝스 압축이 효과적인 경우"를 물으면 → 동일한 데이터가 연속해서 많이 나올 때라고 답하세요.

2. 데이터 압축

허프만 트리(Huffman Tree) 시뮬레이터

학습지 3) canada / murmured 를 허프만 트리로 압축하기 — 빈도수 계산 → 트리 구성 → 문자별 코드 → 전체 코드

🌳 허프만 코딩이란?

문자의 등장 빈도수를 기준으로 이진 트리를 만들어, 자주 나오는 문자는 짧은 코드, 드물게 나오는 문자는 긴 코드를 부여하는 무손실 압축 방식입니다.

  1. 각 문자의 빈도수를 센다
  2. 빈도수가 가장 작은 두 노드를 골라 합쳐 새로운 부모 노드(빈도수 합)를 만든다
  3. 노드가 하나만 남을 때까지 반복한다 → 이것이 허프만 트리
  4. 루트에서 왼쪽=0, 오른쪽=1로 따라가며 각 문자의 코드를 읽는다

Q1 "canada" 허프만 트리

문자 구성: c a n a d a → a가 3번, 나머지는 1번씩

트리를 먼저 생성해보세요. (위 버튼 클릭)

Q2 "murmured" 허프만 트리

문자 구성: m u r m u r e d → m 2번, u 2번, r 2번, e 1번, d 1번

트리를 먼저 생성해보세요. (위 버튼 클릭)

⚠️ 허프만 트리는 빈도수가 같은 노드가 여러 개일 때 묶는 순서에 따라 코드가 달라질 수 있습니다. 시험에서는 보통 "코드 길이의 총합" 또는 "압축 후 전체 비트 수"가 맞으면 정답으로 인정되는 경우가 많으니, 선생님이 제시한 풀이 순서를 함께 확인하세요.

3. 데이터 암호화

데이터 암호화의 정의와 방식

Q 데이터 암호화의 정의와 방식을 설명하시오

📌 정의

데이터 암호화란 평문(원본 데이터)을 특정 규칙(키)을 이용해 제3자가 알아볼 수 없는 형태인 암호문으로 바꾸는 것입니다. 정당한 사용자는 복호화(decryption)를 통해 다시 원래 데이터로 되돌릴 수 있습니다.

🔑 암호화 방식

① 대칭키 암호화 (비밀키 방식)

  • 암호화할 때와 복호화할 때 같은 키를 사용
  • 속도가 빠름
  • 키가 유출되면 보안이 무너짐 → 키 전달이 문제

② 비대칭키 암호화 (공개키 방식)

  • 공개키(암호화)와 개인키(복호화)가 서로 다름
  • 공개키는 누구나 알아도 되지만, 개인키는 본인만 보관
  • 속도는 느리지만 키 분배 문제를 해결

💡 시험 포인트: "공개키로 암호화하면 → 그 쌍이 되는 개인키로만 복호화 가능"이라는 원리, 그리고 두 방식의 속도/보안성 차이를 비교하는 문제가 잘 나옵니다.

4. 데이터 모델링

데이터 모델링 정의와 구조 (표/그래프/트리)

Q1 데이터 모델링의 정의

데이터 모델링이란 현실 세계의 데이터를 일정한 형식과 구조로 표현하여, 컴퓨터가 처리하고 사람이 분석하기 쉽도록 정리하는 과정입니다. 데이터 간의 관계와 속성을 명확히 나타내는 것이 핵심입니다.

Q2 데이터 모델링 구조 3가지: 표 / 그래프 / 트리

📊 표 (Table)

행과 열로 구성되어 데이터를 2차원 격자 형태로 정리. 항목 간 비교가 쉽고, 엑셀·관계형 데이터베이스(RDB)에서 주로 사용.

🔗 그래프 (Graph)

노드(정점)와 간선(엣지)으로 데이터 간의 복잡한 연결 관계를 표현. SNS 친구관계, 지도 네트워크 등에 사용. 순환(cycle) 구조도 가능.

🌳 트리 (Tree)

하나의 루트에서 가지치기 형태로 뻗어나가는 계층 구조. 순환이 없음(비순환). 폴더 구조, 조직도, 허프만 트리 등이 대표적 예시.

💡 비교 포인트: 트리는 그래프의 한 종류(순환이 없는 그래프)라는 점, 표는 관계가 단순할 때, 그래프/트리는 관계가 복잡하거나 계층적일 때 사용한다는 점을 기억하세요.

① 버블 정렬 — 4 2 5 1 3 오름차순 정렬

서로 인접한 값을 계속 비교하며 교환. 아래 버튼으로 한 단계씩 직접 진행해보세요.

전체 비교횟수0
교환횟수0
현재 단계-

정답 최종 비교/교환 횟수

4 2 5 1 3을 버블 정렬로 오름차순 정렬할 때:

전체 비교횟수 = 10 회 (4+3+2+1, n=5일 때 최대 비교횟수)

교환횟수 = 6 회

버블 정렬은 항상 인접한 두 값을 비교하므로, 정렬이 빨리 끝나도 모든 패스를 다 비교하면 비교횟수는 n(n-1)/2로 동일합니다 (단, 중간에 교환이 없으면 조기 종료하는 최적화 버전도 있어요). 위 시뮬레이터로 직접 비교/교환 횟수를 세어 확인해보세요!

전체 비교횟수0
교환횟수0

정답 최종 비교/교환 횟수

4 2 5 1 3을 선택 정렬로 오름차순 정렬할 때:

전체 비교횟수 = 10 회 (4+3+2+1)

교환횟수 = 3 회

선택 정렬은 라운드마다 "최솟값을 찾는 비교"는 매번 일어나지만, "교환"은 이미 제자리에 있으면 생략할 수 있어 버블 정렬보다 교환 횟수가 적은 경우가 많습니다.

5. 정렬 알고리즘

③ 퀵 정렬 — 분할 정복(Divide and Conquer)

하나의 문제를 두 개로 나누어 각각 해결하면 원래 문제가 해결되는 분할 정복 알고리즘. 기준값(피봇)을 정해 피봇보다 작은 값/큰 값으로 분류합니다.

🧩 개념 정리

  1. 배열에서 피봇(pivot)을 하나 정한다 (보통 첫 값, 마지막 값, 또는 중간값)
  2. 피봇보다 작은 값은 왼쪽 그룹, 큰 값은 오른쪽 그룹으로 분류한다
  3. 왼쪽 그룹과 오른쪽 그룹에 대해 각각 같은 과정을 재귀적으로 반복한다
  4. 더 이상 나눌 수 없을 때(원소가 0~1개)까지 반복하면 전체가 정렬된다

Q 4 2 5 1 3 을 퀵 정렬로 오름차순 정렬 (피봇 = 마지막 값 기준)

1단계 피봇=3 → [2,1] / 3 / [4,5]

2단계 왼쪽 [2,1]에서 피봇=1 → [] / 1 / [2] ⟶ 1,2

3단계 오른쪽 [4,5]에서 피봇=5 → [4] / 5 / [] ⟶ 4,5

최종 결과: 1 2 3 4 5

퀵 정렬은 피봇을 어떻게 정하느냐에 따라 분할 과정이 달라질 수 있습니다. 위 풀이는 "각 그룹의 마지막 값을 피봇으로" 정한 한 가지 예시입니다.

6. 파이썬 프로그래밍

📐 파이썬 변수명 규칙

6. 파이썬 프로그래밍

② 연산자의 활용 — 프로그램 결과는?

각 줄을 직접 예측해보고 클릭해서 정답을 확인하세요.

# 산술 연산자

print(100 + 200 / 50)정답 보기
print((100 + 200) / 50)정답 보기
print((100 + 200) // 50)정답 보기
print((100 + 200) % 50)정답 보기

# 비교 연산자

print(10 > 3)정답 보기
print(10 == 3)정답 보기
print(10 != 3)정답 보기

# 논리 연산자

print((30 == 30) and (10 > 20))정답 보기
print((10 > 5) or ((3 ** 2) == 9))정답 보기
print(not (10 == 10))정답 보기

💡 핵심 포인트 풀이

  • / 는 결과가 항상 실수(float) → 100+200/50 은 곱셈/나눗셈이 덧셈보다 우선이라 200/50=4.0, 100+4.0=104.0
  • // 는 몫만 정수로 반환 (소수점 버림), % 는 나머지 반환
  • and 는 둘 다 참이어야 True, or 는 하나만 참이어도 True
  • 3 ** 2 는 거듭제곱(3²=9) → 9==9 는 True → or 전체 결과 True
  • not 은 True/False를 뒤집음
6. 파이썬 프로그래밍

③④ 다음 프로그램의 결과는?

Q3 random.randint 프로그램

import random
num=random.randint(1,10)
print(num)

random.randint(1, 10)은 1부터 10까지(양 끝 포함) 정수 중 하나를 무작위로 반환합니다.

➡️ 정답: 실행할 때마다 1~10 사이의 정수 중 하나가 출력됨 (값이 매번 달라질 수 있음 — 정해진 고정값이 아님)

sum=0

6. 인공지능

1) 지능 에이전트 (Intelligent Agent)

Q 지능 에이전트의 정의

지능 에이전트란 센서(환경 인식)를 통해 주변 환경을 감지하고, 액추에이터(행동 수행)를 통해 그 환경에 작용하는 자율적인 개체(시스템)를 말합니다. 즉, 스스로 상황을 판단하고 목표를 달성하기 위해 행동하는 인공지능 시스템입니다.

Q 지능 에이전트의 4가지 특성

⚡ 반응성 (Reactivity)

환경의 변화를 감지하고 그에 맞게 즉각적으로 반응하는 능력

🧭 자율성 (Autonomy)

사람의 직접적인 개입 없이 스스로 판단하고 행동하는 능력

🎯 목표지향성 / 능동성 (Pro-activeness)

단순히 반응만 하는 게 아니라, 목표 달성을 위해 스스로 행동을 계획·주도하는 능력

🤝 사회성 (Social ability)

다른 에이전트나 사람과 상호작용하고 협력할 수 있는 능력

💡 암기 팁: "반·자·목·사" (반응성-자율성-목표지향성-사회성)로 외우면 편해요. 학습지에는 목표지향성 항목이 비어있을 수 있는데, 교과서 기준 4대 특성 중 하나로 꼭 함께 챙기세요.

Q 지능 에이전트는 어떤 순서로 동작하는가?

① 인식 (Perceive)

센서로 환경 정보를 수집

→
② 판단 (Think/Decide)

수집한 정보를 분석하고 행동을 결정

→
③ 행동 (Act)

액추에이터로 환경에 실제 작용

이 인식 → 판단 → 행동 과정이 한 사이클로 끝나는 것이 아니라, 변화된 환경을 다시 인식하면서 반복(순환)됩니다. 이것이 지능 에이전트가 지속적으로 환경에 적응하는 방식입니다.

6. 인공지능

3) 기계학습 — 정의, 일반 SW와의 차이, 종류 3가지

Q 기계학습의 정의

기계학습(Machine Learning)이란 컴퓨터가 명시적으로 프로그래밍되지 않아도, 주어진 데이터로부터 스스로 규칙(패턴)을 학습하여 새로운 데이터에 대한 예측이나 판단을 수행하는 기술입니다.

Q 일반 소프트웨어와 기계학습의 차이

일반 소프트웨어기계학습
규칙사람이 직접 모든 규칙(조건문)을 작성데이터로부터 규칙을 스스로 학습
입력입력 + 규칙 → 출력입력 + 출력(정답) → 규칙(모델)을 도출
새로운 상황규칙에 없으면 처리 불가학습된 패턴으로 추론·일반화 가능

💡 핵심: 일반 SW는 "규칙 → 결과"를 사람이 짜지만, 기계학습은 "데이터(입력+결과) → 규칙"을 컴퓨터가 스스로 찾아낸다는 점이 가장 큰 차이입니다.

Q 기계학습의 종류 3가지

① 지도학습 (Supervised Learning)

입력 데이터와 정답(레이블)이 함께 주어진 상태에서 학습하여, 새로운 입력에 대한 정답을 예측하는 방식입니다.

예: 고양이/개 사진(입력)+ 정답 라벨 → 분류 모델 학습 / 스팸 메일 분류, 집값 예측 등

② 비지도학습 (Unsupervised Learning)

정답(레이블) 없이 입력 데이터만 주어진 상태에서, 데이터 안의 숨겨진 구조나 패턴, 군집을 스스로 찾아내는 방식입니다.

예: 고객 데이터를 비슷한 특성끼리 그룹으로 묶는 군집화(clustering)

③ 강화학습 (Reinforcement Learning)

에이전트가 환경과 상호작용하며 행동에 대한 보상(reward)을 받고, 보상을 최대화하는 방향으로 행동 전략을 스스로 학습하는 방식입니다.

예: 알파고처럼 게임에서 이기는 행동에는 보상을, 지는 행동에는 패널티를 주며 학습

💡 구분 기준 한 줄 요약: 정답이 있다→지도학습 / 정답 없이 패턴만 찾는다→비지도학습 / 보상을 통해 행동을 학습→강화학습
