II. 데이터  ·  01. 데이터 압축 p.44~49

데이터 압축의 개념과 손실·무손실 압축

학습목표: 디지털 데이터 압축의 개념과 필요성을 설명하고, 압축 방법 간의 효율성을 비교·분석할 수 있다.

① 데이터 압축은 무엇이고 왜 필요할까?

데이터 압축이란 데이터의 크기를 규칙에 따라 줄이는 기술로, 데이터 전송 속도를 개선하고 저장 공간을 절약하여 컴퓨팅 시스템 자원을 효율적으로 활용할 수 있다.

예: 클라우드로 사진을 공유할 때 압축된 사진을 공유하면 저장 공간을 절약하고 업로드·다운로드 시간을 단축할 수 있다. 인터넷 사용량이 제한된 상황에서는 전송 비용도 절감된다.

압축률 공식

압축률 = (원본 데이터 크기 − 압축된 데이터 크기) / 원본 데이터 크기 × 100%

예: 원본 100KB → 압축 후 30KB라면 압축률은 70%. 압축률이 높을수록 데이터 용량이 더 많이 줄었다는 뜻이다.

② 손실 압축 vs 무손실 압축

데이터를 압축하는 방법에는 손실 압축과 무손실 압축이 있다. 무손실 압축은 압축했다가 복원했을 때 원본 데이터와 완전히 일치하지만, 손실 압축은 원본 데이터와 일치하지 않는다.

🔺 무손실 압축

  • 원본 데이터의 모든 정보를 유지하며 압축
  • 비교적 압축률은 낮지만 압축 후 데이터 품질이 높다
  • 복원된 데이터는 원본과 일치
  • 주로 텍스트 파일을 압축하는 데 사용
  • 형식 예시: ZIP, GIF, PNG

🔻 손실 압축

  • 주요 정보는 보존하고, 사람이 인지하기 어려운 일부 정보는 제거하여 압축
  • 비교적 압축률이 높다
  • 압축 시 손실이 발생하여 복원된 데이터는 원본과 일치하지 않음
  • 주로 대용량 동영상·음악을 압축하는 데 사용
  • 형식 예시: JPEG(정지영상), MPEG·MP4(동영상), MP3(음악)

💡 데이터 압축 방법은 데이터의 유형과 사용하는 목적에 따라 결정된다.

③ 문자열 데이터는 어떻게 압축할까?

컴퓨팅 시스템에서 문자열의 반복과 규칙을 이용하면 더 적은 수의 비트로 문자열을 효율적으로 표현할 수 있다. (일반적으로 한 문자당 8비트(1바이트)로 표현한다.)

런-렝스(Run-length) 압축 | 문자열에서 반복되는 문자를 해당 문자와 반복 횟수를 번갈아 가며 표현하는 압축 방법이다.

예: 문자열 'aaabbcccc' 10개 문자(80비트)가 압축 문자열에서는 6개 문자(48비트)로 표현됨 → 압축률 = (80−48)/80×100 = 40%

허프만(Huffman) 압축 | 문자열에서 자주 나타나는 문자에 더 짧은 코드를 할당하고, 드물게 나타나는 문자에 긴 코드를 할당하는 규칙을 갖는 허프만 코드를 사용하는 압축 방법이다.

런-렝스 압축은 동일한 문자가 연속적으로 많이 나타나는 데이터에 효율적이고, 허프만 압축은 연속적이지 않아도 자주 나타나는 문자가 포함된 데이터에 효율적이다. 문자열의 특성에 따라 압축 방식을 선택하는 것이 좋다.

④ 이미지·소리 데이터는 어떻게 압축할까?

파일 형식특징
BMP무압축 방법으로 모든 화소에 색 정보 저장. 품질은 좋으나 파일 용량이 큼
JPEG손실 압축. 색 정보를 줄이거나 일부 이미지 제거하여 압축. 비교적 파일 크기가 작아 웹에서 사용
PNG무손실 압축. 중복 패턴을 찾아 하나의 데이터로 대체. 다양한 색·투명도 지원
GIF무손실 압축. 256색(8비트) 표현 가능. 아이콘·간단한 애니메이션에 적합
파일 형식특징
WAV무압축. 원본 품질 유지, 파일 크기 큼
MP3손실 압축. 사람이 잘 인지하지 못하는 소리 성분 제거. 웹·스트리밍에 적합
WMA손실 압축. 파일 크기 작음, 다양한 비트 전송률 지원
FLAC무손실 압축. 음원 사이트에서 많이 사용, 원본 품질 유지하면서 크기 절감

소리 데이터 품질 비교 요소: 샘플링 주파수(Hz), 비트 깊이, 비트 전송률(Bitrate), 스펙트로그램

런렝스 압축 문제풀이

교과서 활동: banana를 런-렝스 방법으로 압축하여 효율성을 비교해 보자.

활동 문자열 "banana"를 런렝스로 압축하면?

원본: b a n a n a — 같은 문자가 연속하지 않으므로 모든 문자의 반복횟수가 1이다.

➡️ 압축 결과: b1a1n1a1n1a1

압축 전: 6글자 × 8비트 = 48비트

압축 후: (문자+숫자) 6쌍 × 16비트 = 96비트

⚠️ banana는 같은 문자가 연속해서 반복되지 않기 때문에, 런렝스 압축을 적용하면 오히려 원본보다 용량이 2배 커진다. 이것이 바로 이 활동의 핵심 포인트 — 런렝스 압축은 "같은 값이 연속해서 많이 나타날 때"만 효과적이라는 사실을 직접 확인하는 것이다.

예제 HELLO 를 런렝스 압축하면?

원본: H E L L O

문자HELO
반복횟수1121

➡️ 압축 결과: H1E1L2O1

압축 전 5byte → 압축 후 4쌍×2byte = 8byte (오히려 증가)

예제 GOOOOGLE 를 런렝스 압축하면?

원본: G O O O O G L E (O가 4번 연속)

문자GOGLE
반복횟수14111

➡️ 압축 결과: G1O4G1L1E1

압축 전 8byte → 압축 후 5쌍×2byte = 10byte

G가 중간에 끊겨서 다시 등장하기 때문에 효율이 낮다. 만약 GGGGOOOO처럼 큰 덩어리로 반복되면 압축 효과가 커진다.

허프만 트리(Huffman Tree) 시뮬레이터

abracadabra, banana 등 직접 단어를 입력해 허프만 트리·코드를 생성하고 압축률을 확인해 보세요.

🌳 허프만 코딩 5단계

  1. 1단계: 문자열에서 각 문자가 나타나는 빈도수를 계산하고 정렬한다
  2. 2단계: 각 문자와 빈도수를 저장하는 노드(물컵)를 생성한다
  3. 3단계: 빈도수가 가장 작은 두 노드를 결합해 부모 노드를 만든다 (반복)
  4. 4단계: 각 문자에 해당하는 이진 코드를 생성한다 (왼쪽=0, 오른쪽=1)
  5. 5단계: 원본 문자열을 생성된 이진 코드로 변환하여 표현한다

⚠️ 빈도수가 같은 노드가 여러 개일 때는 묶는 기준(예: 알파벳순)을 정해서 진행해야 한다.

🔤 단어를 입력해서 직접 트리를 만들어 보세요

교과서 예제 abracadabra (p.46~47)

문자 구성: a(5), b(2), r(2), c(1), d(1)

문자abrcd
이진 코드01111011001101

➡️ 전체 코드: 0 111 10 1100 0 1101 0 111 10 0

압축 전: 11자×8비트 = 88비트  |  압축 후: 23비트

압축률 = (88−23)/88×100 ≈ 73.9%

교과서 활동 banana (p.50)

문자 구성: b(1), n(2), a(3)

문자bna
이진 코드00011

➡️ 전체 코드: 00 1 01 1 01 1 (9비트)

압축 전: 6자×8비트=48비트 → 압축 후 9비트로, 매우 효율적인 압축

교과서 풀이는 a(3)를 가장 먼저 분리하지 않고 b(1)·n(2)를 묶는 방식도 가능하며, 묶는 순서에 따라 코드 형태는 달라질 수 있다(비트 수 총합은 동일하거나 유사).

II. 데이터  ·  02. 데이터 암호화 p.52~56

데이터 암호화 — 대칭키·비대칭키

학습목표: 암호화의 개념을 이해하여 암호화 및 복호화 과정을 설명하고, 데이터를 안전하게 관리하는 사례를 분석할 수 있다.

① 암호화란 무엇일까?

암호화란 암호화 알고리즘과 암호화 키를 이용해 원본 데이터를 변형된 형태로 바꾸는 보안 기술로, 허락하지 않은 사람에게 암호화된 데이터가 노출되더라도 데이터의 내용을 확인하기 어렵게 하는 기술이다. 반대로 복호화는 암호화된 데이터를 원본 데이터로 복원하는 것을 말한다.

평문(원본 텍스트) → 암호화 키로 암호화 → 암호문 → 복호화 키로 복호화 → 평문

② 암호화 방식의 종류

🔑 대칭키 암호화

암호화와 복호화 과정에서 같은 키를 사용하는 방식. 비밀키 암호화 방식이라고도 한다. 비밀키를 송신자와 수신자가 미리 교환하여 공유하고 있어야 한다.

예: DES 방식

🔐 비대칭키 암호화

암호화와 복호화 과정에서 서로 다른 키를 사용하는 방식. 공개키 암호화 방식이라고도 한다. 공개키는 누구나 알 수 있지만, 공개키로 암호화된 암호문은 특정 개인키로만 복호화할 수 있다.

예: RSA 방식. 활용: 전자서명, 공동인증서, 금융인증서

비대칭키 암호화 방식 톺아보기: ① 송신자는 수신자가 제공한 공개키를 획득 후 데이터 암호화 → ② 암호화된 데이터를 수신자에게 전송 → ③ 수신자는 자신이 지닌 개인키로 복호화

③ 암호 기법에는 어떤 것이 있을까?

치환형 암호 기법 | 알파벳, 기호와 같은 문자와 숫자 등을 다른 문자로 치환하여 암호화하는 방식이다. 문자의 순서에 상관없이 무작위로 치환할 수 있고, 평문의 각 문자가 암호문의 다른 문자와 일대일 대응한다.

대표적 예: 시저 암호 — 로마 제국 황제 카이사르가 사용했다고 알려진 기법. 알파벳을 일정하게 이동한 후 각 문자를 다른 알파벳으로 치환한다. 방법이 간단해서 해독이 쉽다는 단점이 있다.

전치형 암호 기법 | 문자를 재배열하거나, 일정한 패턴에 따라 문자의 위치를 변경하는 등 문자의 순서를 변경하여 원래 메시지의 의미를 숨기는 암호화 방식이다.

전치형 암호는 표(격자)를 그려 가로로 메시지를 채운 뒤 세로로 읽어 암호화하고, 복호화할 때는 반대로 진행한다 — 자세한 풀이는 다음 페이지에서 직접 연습합니다.

④ 데이터 암호화는 어디에 활용될까?

III. 알고리즘과 프로그래밍  ·  01. 문제 분해와 모델링 p.84~91

문제 분해(성분/절차)와 모델링(표/그래프/트리)

학습목표: 복잡한 문제를 해결 가능한 작은 문제로 분해하고 모델링할 수 있다.

① 문제 분해란 무엇일까?

복잡한 문제를 작은 단위의 문제로 나누는 것을 문제 분해라 한다. 문제 분해를 하면 문제를 구성하는 성분을 쉽게 파악할 수 있으며, 절차가 명확하게 드러나 해결 가능한 문제로 접근할 수 있다.

문제 분해의 두 가지 기준:

🧩 성분 분해

문제를 구성하는 전체 구성 요소를 파악하고, 이를 구분하여 소분류로 세분화하는 작업. 분해된 문제는 시간에 종속되지 않는 특징을 가진다.

예: 아침 식사 → 계란 볶음밥 / 사과 주스 → 각각의 재료들

🔢 절차 분해

문제를 해결하는 여러 절차를 파악하고, 이를 구분하여 문제 해결 과정의 순서에 따라 세분화하는 작업. 앞 단계 결과물이 다음 단계 자원으로 쓰이므로 절차를 지키지 않으면 문제가 발생할 수 있다.

예: 아침 식사 → 재료 준비하기 → 야채 썰기 → 재료 볶기 → 간 맞추기 → 완성

💡 문제의 성질에 따라 성분이나 절차로 세분화하거나, 혼합해서 세분화할 수 있다(성분 분해 후 절차로 분해, 또는 절차 분해 후 성분으로 분해).

② 모델링이란 무엇일까?

문제를 효율적으로 해결하기 위해서는 문제를 이해하고 분석하여 핵심 요소를 추출하는 추상화 과정이 필요하다. 추상화를 하면 복잡한 문제를 단순화할 수 있으며, 문제 해결에 적용할 수 있는 규칙을 발견하거나 모델링을 할 수 있다.

모델링이란 현실 세계의 복잡한 문제를 이해하기 쉬운 구조로 표현하는 과정을 말한다. 컴퓨터 과학에서의 모델링은 이러한 과정으로 얻어진 결과를 모델이라 하며, 컴퓨팅 시스템이 처리할 수 있는 형태로 만드는 것을 말한다.

데카르트가 파리의 위치를 (x, y) 좌표로 표현한 것처럼, 모델링은 추상화 과정을 통해 추출된 요소들의 관계를 쉽게 표현하기 위해 표, 그래프, 트리 등 다양한 형태로 구조화한다.

③ 모델링 구조 — 표 / 그래프 / 트리

📊 표

행과 열을 이용하여 데이터를 구조화하는 방법. 데이터를 효율적으로 검색·관리할 수 있는 장점이 있다.

🔗 그래프

데이터를 노드로 표시하고, 데이터와 데이터를 연결하는 간선으로 구성. 연결선으로 방향(또는 무방향)을 표현할 수 있다. 최단 거리 탐색 등에 사용.

🌳 트리

데이터 사이의 관계를 계층적으로 표현한 구조. 부모 노드와 자식 노드로 구성되며, 하나의 루트 노드에서 시작한다. 가족관계도, 책 목차, 폴더 구조 등.

④ 분할 정복 알고리즘

분할 정복 알고리즘(Divide and Conquer Algorithm)은 복잡한 문제를 해결 가능한 작은 문제로 나누어 해결한 후, 작은 문제의 해답을 모아서 원래의 문제를 해결하는 기법이다.

  1. 1단계: 큰 문제를 작은 문제로 나누기 (분할)
  2. 2단계: 작은 문제 해결하기
  3. 3단계: 작은 문제의 해를 합치기 (결합)

예: 놀이공원에서 출입구→유령의 집→관람차→회전 그네→신나는 계곡까지의 최단 경로를 구간별로 나누어 각각 최단 경로를 찾은 뒤 합쳐서 전체 최단 경로를 구한다.

③ 퀵 정렬 — 분할 정복(Divide and Conquer)

하나의 문제를 두 개로 나누어 각각 해결하면 원래 문제가 해결되는 분할 정복 알고리즘의 한 종류. 데이터에서 기준값(피봇)을 정하여 피봇보다 작은 값과 큰 값으로 분류하여 정렬한다. 수행 시간은 빠르지만 이미 정렬된 데이터라면 수행 시간이 매우 느리다.

📐 퀵 정렬 동작 원리

  1. 데이터 중에서 첫 번째 값을 선택하여 피봇으로 정한다
  2. 피봇을 중심으로 피봇보다 작은 수는 왼쪽으로 옮기고, 피봇보다 큰 수는 오른쪽으로 옮긴다
  3. 피봇을 제외한 왼쪽 값과 오른쪽 값에 대해 ①과 ②를 수행한다(재귀)

피봇을 정하는 방법은 다양하다: 첫 번째/마지막 값을 피봇으로, 무작위 피봇 선택, 첫 번째·중앙값·마지막 값 중 중앙값(median)을 피봇으로 선택 등. 교과서는 첫 번째 값을 피봇으로 사용한다.

Q 4 2 5 1 3 을 퀵 정렬로 오름차순 정렬 (피봇 = 첫 번째 값)

1단계 피봇=4(첫 번째 값) → 2(2<4, 왼쪽) → 5(5>4, 오른쪽) → 1(1<4, 왼쪽) → 3(3<4, 왼쪽) ⟶ [2,1,3] / 4 / [5] (비교횟수 4)

2단계 왼쪽 그룹 [2,1,3]에서 다시 피봇=2(첫 번째 값) → 1(1<2, 왼쪽) → 3(3≥2, 오른쪽) ⟶ [1] / 2 / [3] (비교횟수 2)

3단계 오른쪽 그룹은 5 하나만 있어 정렬 종료. 왼쪽 그룹의 1, 3도 각각 1개뿐이므로 종료.

최종 결과: 1 2 3 4 5

총 비교횟수 = 6회 (1단계 4회 + 2단계 2회)

알고리즘의 성능은 어떻게 평가할까? — 빅오(Big-O) 표기법

알고리즘의 효율성을 상한선(최악) 기준으로 표기하는 방법.

⏱️ 시간 복잡도와 공간 복잡도

시간 복잡도

순차 탐색과 이진 탐색

학습목표: 탐색 알고리즘의 개념을 이해하고, 이를 활용하여 데이터를 탐색하며 수행 과정을 비교·분석할 수 있다.

① 탐색 알고리즘이란?

주어진 데이터에서 원하는 값을 찾는 과정을 탐색이라고 한다. 컴퓨팅 시스템에서는 원하는 데이터를 효율적으로 찾기 위해 탐색 알고리즘을 사용한다. 대표적인 예로는 순차 탐색, 이진 탐색 등이 있다.

정렬 알고리즘은 데이터를 일정한 순서대로 나열하는 방법이고, 탐색 알고리즘은 저장된 데이터 중에 원하는 것을 효율적으로 찾는 방법이다.

② 순차 탐색

처음부터 차례대로 비교하면서 탐색하는 방법. 데이터가 정렬되어 있지 않아도 수행할 수 있지만, 찾는 데이터가 없거나 마지막에 있으면 전체 데이터의 개수만큼 비교해야 한다.

  1. 첫 번째 데이터를 선택한다
  2. 선택한 데이터와 찾는 데이터를 비교한다 — 같으면 탐색 종료, 같지 않으면 다음 값 선택
  3. 비교할 데이터가 남아 있는 동안 ②를 반복한다

예: 6장의 카드 [2,5,1,6,4,5]에서 1을 순차 탐색하면 3번째 비교에서 찾는다 (2 vs 1, 5 vs 1, 1 vs 1 → 종료).

③ 이진 탐색

정렬된 데이터에서 탐색 범위를 반으로 줄여 가며 원하는 값을 찾는 알고리즘. 숫자 맞히기 게임을 생각하면 이해하기 쉽다.

  1. 탐색 범위를 정한다
  2. 중간 위치(mid)를 구한다: mid = (left + right) // 2
  3. mid의 데이터와 찾으려는 데이터를 비교한다 — 같으면 종료, mid값이 더 크면 처음부터 mid−1까지를 탐색 범위로, mid값이 더 작으면 mid+1부터 마지막까지를 탐색 범위로 정한다
  4. 찾는 데이터가 탐색 범위에 있는 동안 ②③ 과정을 반복한다

예: 10장의 카드 [1,4,6,9,10,24,31,42,50,52]에서 52를 찾으면 — mid=10(5번째)→10<52→오른쪽, mid=42(8번째)→42<52→오른쪽, mid=50(9번째)→50<52→오른쪽, mid=52(10번째)→일치! 4단계만에 탐색 완료.

표준 입출력과 파일 입출력

학습목표: 표준 입출력과 파일 입출력을 활용한 프로그램을 작성할 수 있다.

① 표준 입출력이란?

표준 입력이란 컴퓨팅 시스템 장치를 이용해 사용자로부터 직접 값을 입력받는 것(대부분 키보드·마우스). 표준 출력은 프로그램에서 자료를 출력하는 것(대부분 모니터). 파이썬의 표준 입력 함수는 input(), 표준 출력 함수는 print()이다.

표준 입력 input()표준 출력 print()
사용자로부터 데이터를 한 줄씩 입력받는다. 숫자나 문자 등을 입력받을 수 있으며 자료형은 문자열로 저장된다.괄호 안의 내용을 화면에 출력한다. 변수, 문자열 등을 함께 출력할 수 있다.

⚠️ input()으로 입력받은 값은 항상 문자열형(str)이다. 10000을 입력해도 실제로는 문자열 '10000'으로 처리되어 계산을 할 수 없다 → 숫자형으로 변환(int()) 후 사용해야 한다.

리스트, 인덱스, 슬라이싱, 다차원 리스트

학습목표: 다차원 데이터의 개념을 설명하고, 이를 활용한 프로그램을 작성할 수 있다.

① 리스트란?

컴퓨팅 시스템에서 다양한 자료들을 순서대로 저장하는 것을 리스트라 한다. 변수와 비슷한 개념이지만 여러 개의 값을 동시에 저장할 수 있는 것이 가장 큰 차이점이다.

book = ['홍길동전', '김한국', 2022, '813.5-아22ㄱ-8', True]
# 인덱스:    [0]        [1]      [2]         [3]              [4]

인덱스란 요소가 저장된 위치의 번호. 리스트 길이가 n이면 인덱스는 0부터 n−1까지의 범위를 가진다. 음수 인덱스로 끝에서부터(-1부터 거꾸로) 접근할 수도 있다.

① 조건문 (if-else, elif)

주어진 조건이 참인지 거짓인지에 따라 다른 명령을 처리하도록 하는 것을 조건문이라고 한다.

if 조건:

else:

조건이 참(True)이면 명령 블록1, 거짓(False)이면 명령 블록2를 실행한다. else가 없으면 조건이 거짓일 때 아무 것도 실행되지 않는다.

if 조건1:

elif 조건2:

else:
    명령 블록3

조건이 하나가 아니고 여러 개일 경우 elif문을 사용해 조건을 추가로 입력할 수 있다. 조건문 안에 조건문이 있는 구조는 중첩 조건문이라 한다.

④ 중첩 반복문

반복문 안에 여러 개의 반복문을 포함하는 것을 중첩 반복문이라고 한다. 한 개의 반복문으로 구현되지 않는 복잡한 프로그램은 중첩 반복문을 사용해서 해결할 수 있다.

    pick = random.randint(1,5)  # 1~5반 중 한 반 뽑고 pick 변수에 저장

① 지능 에이전트란 무엇일까?

컴퓨터 과학 분야에서 인간 대신 특정 역할을 수행하는 자동화 시스템을 에이전트(agent)라고 한다. 지능 에이전트(intelligent agent)란 센서를 통해 주변 환경을 인식하고, 스스로 판단하여 목적에 맞게 작업을 수행하는 시스템을 말한다.

지능 에이전트는 목표를 가지고 사람이나 환경과 상호 작용을 통해 상황에 따라 적절한 행동을 취하며 반응성, 자율성, 사회성의 특성을 지닌다.

② 지능 에이전트의 3가지 특성

⚡ 반응성

환경을 인지하여 실시간으로 반응한다.

AGI(인공일반지능)란 인간이 할 수 있는 모든 지적 업무를 수행할 수 있는 수준의 범용성을 지닌 인공지능을 의미한다. 반면 일반적인 지능 에이전트는 특정 용도에 맞춰 개발된 경우가 많다.

⑤ 인간과 인공지능의 바람직한 관계

인공지능 개발 초창기에는 인간이 정해 놓은 규칙에 따라 인공지능이 주어진 업무를 수행하는 일방적인 관계였다. 기술 발달로 인공지능은 학습을 통해 스스로 규칙을 발견하고 의사 결정을 내리며, 미래 데이터를 예측하고 유용한 통찰력을 제공하는 등 상호 보완적 협력 관계로 변화되었다.

일방적 관계상호 보완적 협력 관계
인공지능은 인간이 사전에 프로그래밍한 규칙대로 작동. 인간이 해야 하는 반복적인 작업을 대체함.인간은 인공지능이 학습할 데이터를 제공하고 훈련시킴. 인공지능 모델은 패턴을 학습하여 새로운 정보와 예측한 미래 데이터를 제공함.

⚠️ 단, 인공지능 기술은 아직 발달 중이며 완벽하지 않음을 인식해야 한다. 특히 인간의 감성과 윤리에 관련된 가치 판단에는 한계가 있다. 인공지능에 절대적으로 의존하기보다는 인공지능을 인간의 능력을 한 단계 높이는 조력자로 간주하고 목적에 맞게 활용해야 한다.

IV. 인공지능  ·  02. 기계학습의 개념과 유형 p.170~174

기계학습의 정의와 일반 소프트웨어와의 차이

학습목표: 기계학습의 개념과 필요성을 설명하고, 지도 학습과 비지도 학습의 차이를 비교·분석할 수 있다.

① 기계학습이란 무엇일까?

기계학습(Machine Learning)은 인공지능의 한 분야로, 데이터에서 규칙을 찾아 스스로 학습하여 목적에 맞는 모델을 생성하고 작업을 수행한다. 기계학습 모델은 학습한 데이터를 바탕으로 예측하고, 특정 기준에 따라 분류하거나 유사한 데이터끼리 묶을 수 있다.

② 일반 소프트웨어와 기계학습의 차이점

일반 소프트웨어는 데이터를 입력받으면 사람이 만든 규칙에 따라 연산하고 결과를 출력한다. 기계학습은 입력된 데이터를 바탕으로 스스로 학습하여 패턴이나 규칙을 찾아 필요한 작업을 수행한다.

일반 소프트웨어기계학습
흐름데이터 입력 → 규칙(프로그램) → 출력훈련 데이터 → 학습 → (데이터 입력 → 출력)
규칙의 출처사람이 직접 작성데이터로부터 스스로 학습

예: 얼룩말 판별 — 일반SW는 "갈기가 있는가? → 발굽이 있는가? → 줄무늬가 있는가?"처럼 사람이 짠 규칙(if문)으로 판별. 기계학습은 얼룩말의 다양한 모습을 학습한 모델이 새 이미지를 보고 스스로 판별.

기계학습은 사물 인식 기술, 자연어 처리(NLP) 기술, 추천 시스템 등에 활용된다.

③ 기계학습은 데이터를 어떻게 학습할까? — 지도 학습 / 비지도 학습 / 강화 학습

기계학습은 데이터를 학습하는 방법에 따라 크게 지도 학습, 비지도 학습으로 나뉜다. 이외에도 데이터가 직접 제공되지 않지만 환경과 상호 작용하여 얻은 정보로 행동을 수정하여 목표에 도달하는 강화 학습이 있다.

📌 지도 학습 (Supervised Learning)

데이터에 정답(레이블)을 함께 제공하여 기계학습 모델을 학습시키는 방법. 레이블이 포함된 데이터를 학습하여 새로운 데이터를 예측하거나 데이터가 어떤 종류에 해당하는지 분류한다.

학습 과정에 사용되는 데이터를 훈련 데이터, 모델의 성능을 평가하는 데 사용되는 데이터를 테스트 데이터라고 한다.

📌 비지도 학습 (Unsupervised Learning)

데이터에 정답을 제공하지 않고 기계학습 모델을 학습시키는 방법. 레이블이 없는 데이터의 특성을 분석하여 서로 비슷한 특성을 가진 데이터끼리 군집화하거나, 데이터가 가진 여러 속성 사이의 연관된 규칙을 찾는다.

📌 강화 학습 (Reinforcement Learning)

에이전트가 환경과 상호 작용하면서 보상을 통해 최적의 행동을 학습하는 방법. 시험에서 좋은 성적을 거뒀을 때 선생님께 칭찬받으면 더 열심히 공부하는 것처럼, 보상을 통해 행동을 강화한다.

지도 학습비지도 학습
개념데이터와 레이블 사이의 관계를 학습해 새 데이터를 분류·예측데이터의 구조나 패턴을 발견하거나 그룹으로 분류
훈련 데이터의 레이블있음없음
활용 예사물 인식, 스팸 메일 분류, 가격 예측데이터 군집화, 추천 시스템

💡 구분 한줄 요약: 정답이 있다→지도학습 / 정답 없이 패턴만 찾는다→비지도학습 / 보상을 통해 행동을 학습→강화학습

🏷️ 데이터 라벨러란?

데이터 라벨러(Data labeler)는 이미지·음성·영상·텍스트 등 다양한 형식의 데이터에 레이블을 붙여 주는 작업을 수행하는 전문가나 업무 수행자. 이 작업을 데이터 라벨링(Data labeling)이라고 한다. 데이터 라벨링이 정확히 이루어지지 않으면 기계학습 모델의 정확도가 떨어져 예측이 부정확해진다.

IV. 인공지능  ·  03. 기계학습과 문제 해결 p.176~181

기계학습의 유형 — 회귀·분류·군집

학습목표: 기계학습을 활용하여 해결할 수 있는 문제와 그렇지 않은 문제를 구분하고, 사회 문제 해결에 기계학습을 적용할 수 있다.

① 기계학습으로 해결할 수 있는 문제 구분하기

기계학습은 대용량의 데이터를 수집할 수 있는 특정 분야에서 효율적으로 활용된다. 정형 데이터는 물론, 이미지·영상 등 비정형 데이터도 기술 발전으로 기계학습에 활용되고 있다.

기계학습으로 해결하기 적합한 문제를 구분할 때 고려할 사항:

⚖️ 데이터 편향성

기계학습 모델을 학습시키는 데 사용하는 데이터에 인간의 특정 편향이 적용되면 왜곡된 판단을 하는 모델을 만들 수 있다. 예: 범죄 발생 가능성에 대한 인종 편향, 특정 언어 중심 학습으로 인한 문화적 편향.

💰 비용 효율성

기계학습 모델을 개발하는 과정에서 데이터 수집·가공, 고성능 시스템 사용 등 많은 비용이 발생한다. 기존 소프트웨어 사용과 기계학습 개발 중 어느 것이 더 비용 면에서 효율적인지 고려해야 한다.

📌 GIGO(Garbage In, Garbage Out): 학습 데이터의 질이 곧 모델 성능을 좌우한다는 원칙 — 대량의 데이터, 양질의 데이터가 모두 중요하다.

② 기계학습의 대표 유형: 회귀, 분류, 군집

📈 회귀(Regression)

이미 알고 있는 입력과 출력의 쌍으로 이루어진 데이터를 활용하여 학습한 후, 연속성을 띠는 새로운 데이터값을 예측하는 방법.

🏷️ 분류(Classification)

레이블이 포함된 데이터를 학습하여 만든 분류 모델에 새로운 입력값이 들어왔을 때 정해진 클래스 중 하나로 나누는 방법.

🔮 군집(Clustering)

레이블이 없는 데이터를 학습하여 숨겨진 패턴을 발견하고, 비슷한 특성을 지닌 데이터끼리 묶어주는 방법.

예제 회귀 — 월별 구독자 수 예측 (p.178)

운영 기간구독자 수
1개월19
2개월39
3개월42
4개월55
5개월70
6개월?

5개월간의 데이터를 점으로 찍고, 모든 점을 동시에 지나는 직선이 없으므로 어느 정도의 추세를 나타내는 선을 그어 추세선에서 6개월의 구독자 수를 예측할 수 있다. 이처럼 선을 그려 새로운 데이터를 예측하는 방법을 선형 회귀(Linear Regression)라고 한다. 여러 요인(지역, 방 개수, 면적 등)으로 하나의 값을 예측하면 다중 선형 회귀라 한다.

예제 분류 — 대기 오염 상태 분류 (p.179)

일차미세 먼지초미세 먼지대기 오염 상태
14115좋음
2275좋음
3110965나쁨
327026?

미세 먼지·초미세 먼지 농도에 따른 대기 오염 상태를 분류 모델로 학습시키면, 클래스를 구분하는 경계를 학습하여 새로운 데이터를 분류한다. 레이블은 개별 데이터의 식별 정보(정답), 클래스는 데이터를 분류하는 데 사용되는 카테고리를 의미한다.

대표적 분류 모델: SVM(서포트 벡터 머신)은 클래스를 나누는 경계선을 그어 구분. KNN(K-Nearest Neighbor)은 가장 가까운 k개의 점을 기반으로 가장 많은 클래스로 데이터를 분류.

예제 군집 — 쇼핑몰 소비자 구매 패턴 분석 (p.181)

온라인 쇼핑몰을 이용하는 고객들의 구매 제품 평균 가격과 구매 횟수 데이터를 군집 모델로 분석하면, 유사도에 따라 고객을 여러 그룹으로 나눌 수 있다. 그 결과 각 고객 그룹에 적합한 맞춤형 홍보 전략을 사용할 수 있다.

k-means는 군집 알고리즘의 하나로, 데이터를 사용자가 지정한 k개의 군집으로 유사도에 따라 분리한다.

🔍 분류 vs 군집, 무엇이 다를까?

분류와 군집 모두 데이터를 나누는 일이라 차이가 모호하게 느껴질 수 있다. 여러 클래스 중 더 높은 확률로 예측되는 곳에 데이터를 할당하는 것을 분류라 한다면, 군집은 레이블이 없는 데이터를 비슷한 특성을 가진 그룹으로 묶는 방법이다.

예: 스팸 메일인지 정상 메일인지 파악하는 것은 클래스 중 하나로 데이터를 할당하는 일이므로 분류. 소비자의 구매 이력을 파악하여 구매 패턴이 비슷한 유형의 고객끼리 나누는 것은 레이블 없는 데이터를 그룹으로 묶는 것이므로 군집.
