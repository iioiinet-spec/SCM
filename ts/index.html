typedef란 무엇인가
01typedef의 역할 — 자료형에 별명 붙이기
typedef는 기존 자료형에 새 이름(별명)을 붙이는 기능입니다. 구조체와 함께 쓰면 매번 struct를 쓰는 불편함을 없앨 수 있습니다.
typedef 없이
struct Student {
    char name[20];
    int age;
};
struct Student s1; // 불편
typedef 사용
typedef struct {
    char name[20];
    int age;
} Student;
Student s1; // 편리!
차이: struct Student s1; → Student s1;
구조체를 일반 자료형처럼 사용 가능!
02typedef struct 해부하기
코드 한 줄씩 분해해 보면:
C — 각 부분의 역할
typedef struct Student {   // ① typedef + 구조체 원래 이름
    char name[20];            // ② 멤버들
    int  age;
} Student;                  // ③ 새 별명 (이게 핵심!)
① struct Student
구조체 원래 이름
② { ... }
멤버 정의
③ } Student;
새 이름 ← 핵심!
03두 가지 작성 방식 비교
방식 A — 원래 이름 있음
typedef struct Student {
    char name[20];
    int age;
} Student;
struct Student s1; 또는 Student s1;
방식 B — 별명만 (더 간단)
typedef struct {
    char name[20];
    int age;
} Student;
Student s1; 만 사용
초보자에게는 방식 B(별명만)가 더 간단하고 이해하기 쉽습니다.
04비유로 이해하기
긴 이름에 짧은 별명을 붙이는 것과 같습니다.
현실
김민수학생대표1반
→ 별명: 민수
코드
struct Student
→ 별명: Student
05마무리 예제 — Book 구조체
C — 전체 코드
#include <stdio.h>
#include <string.h>

typedef struct {
    char title[30];
    int  price;
} Book;

int main() {
    Book b1;
    strcpy(b1.title, "CProgramming");
    b1.price = 18000;

    printf("책 제목: %s\n", b1.title);
    printf("가격: %d원\n", b1.price);
    return 0;
}
책 제목: CProgramming
가격: 18000원
typedef struct 빌더 — 템플릿 선택 후 값 입력, 구조체 메모리 시각화
값을 입력하고 실행 버튼을 누르세요
typedef 유무 코드 비교기 — 선택한 구조체로 두 방식을 나란히 비교
자주 하는 실수 4가지
실수 1닫는 중괄호 뒤 세미콜론 빠뜨리기
typedef struct { int age; } Student ← ; 없음
typedef struct { int age; } Student; ← ; 필수
typedef 구조체는 선언문이므로 마지막에 반드시 세미콜론이 있어야 합니다.
실수 2문자열 멤버에 = 대입 사용
s1.name = "Minsu"; ← char 배열에 = 대입 불가
strcpy(s1.name, "Minsu"); ← strcpy 사용
char 배열은 포인터가 아니므로 = 으로 직접 대입할 수 없습니다. strcpy(string.h)를 사용하세요.
실수 3typedef 이름을 변수로 착각
typedef struct { int x; int y; } Point;
← 여기서 Point는 자료형 이름, 변수가 아님
Point p1; ← 변수는 따로 선언해야 함
p1.x = 3; p1.y = 7;
typedef 뒤의 이름은 자료형 이름입니다. 변수를 만들려면 별도로 선언해야 합니다.
실수 4멤버 접근에서 . 빠뜨리기
p1 x = 3; ← . 없음
p1.x = 3; ← . 연산자 필수
구조체 변수와 멤버 이름 사이에는 반드시 점(.) 연산자가 있어야 합니다.
