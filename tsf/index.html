한 줄 정의

"main에서 만든 구조체 변수를 다른 함수로 보내서 출력·수정·계산에 활용하는 것"

왜 쓰는가?

  • 코드를 역할별로 나눌 수 있다
  • 한 함수가 한 가지 일만 맡는다
  • 재사용이 쉬워진다

실제 쓰임새

  • printStudent() — 출력 전담
  • changeScore() — 값 수정
  • getArea() — 계산 후 반환

핵심 흐름

typedef struct로 구조체 만들기
↓
구조체 변수 선언 및 값 저장
↓
함수에 구조체 전달 (복사 or 주소)
↓
함수 안에서 .또는 -> 로 멤버 접근

방법 1 — 복사 전달

void func(Student s)
  • 구조체 전체가 복사됨
  • 함수 안 s는 복사본
  • 원본이 안 바뀜
  • 호출: func(s1);
  • 멤버 접근: s.score

방법 2 — 주소 전달

void func(Student *s)
  • 구조체 주소만 전달됨
  • 원본에 직접 접근
  • 원본이 바뀜
  • 호출: func(&s1);
  • 멤버 접근: s->score

비유로 이해하기

복사 전달
학생 정보가 적힌 종이를 복사해서 전달. 복사본에 적어도 원본은 그대로.
주소 전달
원본 종이가 있는 위치를 알려줌. 그 위치에서 직접 수정 가능.

언제 어떤 방법을?

  • 출력·읽기만 할 때 → 복사 전달도 충분
  • 원본 값을 수정해야 할 때 → 주소 전달
  • 구조체가 클 때 → 주소 전달이 더 효율적

복사 전달일 때 메모리 상태

함수가 호출되면 s1의 복사본 s가 생깁니다. 함수 안에서 s.score를 바꿔도 원본 s1은 그대로입니다.

main — s1 (원본)

name
"Jisu"
score
80
복사

changeScore() — s (복사본)

name
"Jisu"
score
100 ← 바뀜

함수 종료 후 원본

s1.score
80 (그대로)

주소 전달일 때 메모리 상태

포인터 s는 s1의 주소를 가리킵니다. s->score를 바꾸면 원본 s1.score가 직접 바뀝니다.

main — s1 (원본)

score (before)
80
score (after)
100 ← 직접 변경!
가리킴

changeScore() — *s (포인터)

s
&s1 주소
s->score
= 100 (원본에)

학생 정보 출력 — 복사 전달로 충분

출력만 하므로 원본을 바꿀 필요 없습니다. 값을 그냥 복사해서 전달합니다.

void printStudent(Student s) { printf("이름: %s\n", s.name); // s.멤버 — 점(.) 사용 printf("나이: %d\n", s.age); } int main() { Student s1; strcpy(s1.name, "Minsu"); s1.age = 14; printStudent(s1); // 그냥 전달 return 0; }

실행 결과

이름: Minsu 나이: 14

복사의 함정 — 함수 안에서 바꿔도 원본은 그대로!

changeScore(s1)을 호출하면 s1의 복사본이 함수로 들어갑니다. 함수 안에서 score를 100으로 바꿔도, main의 s1.score는 80 그대로입니다.

void changeScore(Student s) { s.score = 100; printf("함수 안: %d\n", s.score); // 100 } int main() { Student s1; s1.score = 80; changeScore(s1); printf("main: %d\n", s1.score); // 80 (그대로!) return 0; }

실행 결과

함수 안: 100 main: 80

주소 전달 — 원본을 직접 수정

changeScore(&s1)로 주소를 보냅니다. 함수 안의 *s는 s1을 직접 가리키므로, s->score = 100이 원본에 반영됩니다.

void changeScore(Student *s) { s->score = 100; // -> 사용! } int main() { Student s1; s1.score = 80; changeScore(&s1); // 주소 전달 printf("바뀐 점수: %d\n", s1.score); // 100! return 0; }

실행 결과

바뀐 점수: 100
모드를 선택하고 시작 버튼을 누르세요.

점(.) — 일반 구조체 변수

Student s1; s1.score = 90; s1.name // ← .사용
  • s1은 구조체 그 자체
  • 직접 점으로 멤버 접근

화살표(->) — 구조체 포인터

Student *s; s->score = 90; s->name // ← ->사용
  • s는 주소를 담은 포인터
  • ->가 주소를 따라 멤버 접근

->의 정체

s->score 는 (*s).score 와 완전히 동일합니다.

s->score = 100; // 짧고 읽기 쉬움 (*s).score = 100; // 풀어 쓴 것 — 같은 의미

예제 3 (주소 전달) 코드 해설:

실수 1 — 주소가 필요한데 그냥 보내기

틀림
changeScore(s1); // *s 함수에 그냥 전달
맞음
changeScore(&s1); // &로 주소 전달

실수 2 — 포인터인데 . 사용하기

틀림
s.score = 100; // s가 포인터면 오류
맞음
s->score = 100; // 포인터엔 ->

실수 3 — 일반 변수인데 -> 사용하기

틀림
s1->score = 100; // s1은 일반 변수
맞음
s1.score = 100; // 일반 변수엔 .
