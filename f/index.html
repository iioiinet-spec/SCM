Call by Value vs Call by Reference

함수를 호출하면 인자(argument)의 값이 매개변수(parameter)로 복사되어 이동합니다. 아래 시나리오를 고르고 한 단계씩 진행하면, 값(또는 주소)이 스택 프레임 사이를 날아가서 복사되는 순간과, 포인터가 원본을 가리키는 화살표를 눈으로 확인할 수 있습니다.

학습 포인트

      

        1. 값 전달 (call by value) — 인자의 값을 복사해 매개변수에 넣습니다. 함수 안에서 매개변수를 바꿔도 원본은 절대 변하지 않습니다. 그래서 시나리오 1의 swap은 실패합니다.

        2. 참조 전달 (call by reference) — C에는 진짜 참조가 없어서, 대신 주소(&x)를 복사해 넘깁니다. 포인터 매개변수가 원본이 사는 곳을 가리키므로 *a = ...로 원본을 직접 수정할 수 있습니다.

        3. 사실 C는 모든 전달이 복사 — 포인터를 넘길 때도 "주소값"이 복사됩니다. 복사되는 것이 데이터냐, 주소냐의 차이일 뿐입니다.

        4. 선택 기준 — 읽기만 하면 값 전달 + return(시나리오 3)이 깔끔하고, 원본을 바꾸거나 결과를 2개 이상 돌려줘야 하면 포인터를 씁니다.

        5. 배열은 특별하다 — 배열 이름은 전달 순간 첫 원소의 주소로 바뀝니다(decay). int arr[] 매개변수는 사실 int *arr과 같으므로, 함수 안의 수정이 항상 원본에 반영됩니다. 크기는 함께 전달되지 않아 n을 따로 넘기고, 원본 보호가 필요하면 const를 붙입니다.

        6. 포인터 연산 — p + i는 i바이트가 아니라 i \u00d7 sizeof(자료형)만큼 이동합니다 (int면 4바이트씩). 그리고 arr[i]는 *(arr + i)와 완전히 같은 표현 — 배열 인덱싱의 정체는 포인터 연산입니다.

        7. 2차원 배열 — 메모리에는 행 우선(row-major)으로 연속 저장됩니다. 전달 시 int (*m)[열] 포인터로 decay되며, m[i][j]의 위치를 시작 + (i\u00d7열개수 + j)\u00d74로 계산해야 하므로 열 개수는 생략 불가입니다.
