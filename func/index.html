call by value vs call by reference
03call by value값을 복사해서 전달
함수에 변수를 넘기면 값이 복사되어 전달됩니다. 함수 안에서 매개변수를 바꿔도 원본은 그대로입니다.
C — call by value (swap 실패 예시)
void swap(int x, int y) {
    int temp = x;
    x = y;
    y = temp;
    // x, y만 바뀜. a, b는 그대로!
}
int main() {
    int a = 3, b = 7;
    swap(a, b);
    printf("%d %d\n", a, b); // 3 7 (바뀌지 않음)
}
출력: 3 7 ← 원본 a, b 변경 없음
04call by reference 방식주소를 전달 (포인터 사용)
포인터로 원본 주소를 전달하면 함수 안에서 원본을 직접 변경할 수 있습니다.
C — 포인터로 swap 성공
void swap(int *x, int *y) {
    int temp = *x;
    *x = *y;
    *y = temp;
    // *x = a 자체, *y = b 자체
}
int main() {
    int a = 3, b = 7;
    swap(&a, &b);
    printf("%d %d\n", a, b); // 7 3 (교환 성공!)
}
출력: 7 3 ← 원본 a, b 교환 성공
05포인터로 여러 결과 반환
C의 return은 값 하나만 반환합니다. 포인터를 사용하면 함수에서 여러 결과를 동시에 얻을 수 있습니다.
C — 합과 차 동시 반환
void calculate(int a, int b,
               int *sum, int *diff) {
    *sum  = a + b;
    *diff = a - b;
}
int main() {
    int x=20, y=5, s, d;
    calculate(x, y, &s, &d);
    printf("합=%d 차=%d\n", s, d);
}
출력: 합=25 차=15

핵심 비교표
구분 call by value call by reference 방식
전달 방식값 복사주소 전달 (&a)
매개변수int xint *p
원본 변경불가가능 (*p = 값)
호출 방법func(a)func(&a)
사용 시기계산·반환만 필요swap·다중 반환
자주 하는 실수 3가지
실수 1call by value원본이 바뀔 거라는 착각
void change(int x) { x = 50; }
→ 이렇게 하면 원본은 절대 바뀌지 않습니다
void change(int *p) { *p = 50; }
change(&a); ← &a로 주소를 넘겨야 원본이 바뀜
실수 2포인터주소를 안 넘기는 실수
changeValue(a); ← 값을 넘겨서 원본 변경 불가
changeValue(&a); ← &a로 주소를 넘겨야 함
포인터 매개변수(int *p)를 받는 함수를 호출할 때는 반드시 &a 처럼 주소를 전달해야 합니다.
실수 3포인터역참조(*) 빠뜨리기
void change(int *p) { p = 100; } ← p 자체를 바꾸는 이상한 코드
void change(int *p) { *p = 100; } ← *p로 원본에 접근해야 함
p는 주소를 담는 포인터 변수입니다. 원본 값을 바꾸려면 * 역참조 연산자가 반드시 필요합니다.
참고배열은 왜 원본이 바뀌는가
배열을 함수에 전달하면 배열의 시작 주소가 전달됩니다. 그래서 함수 안에서 원소를 바꾸면 원본 배열이 바뀝니다.
C — 배열 전달
void changeArr(int arr[]) {
    arr[0] = 999;   // 원본 배열 변경됨!
}
int main() {
    int a[3] = {1,2,3};
    changeArr(a);
    printf("%d\n", a[0]); // 999
}
arr[]는 내부적으로 포인터처럼 동작합니다. 배열 이름은 첫 번째 원소의 주소이기 때문입니다.
