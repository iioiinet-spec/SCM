허프만 압축 인터렉티브 강의자료

빈도표 → 최소 빈도 2개 합치기 → 허프만 트리 → 코드 생성 → 압축과 복원까지 직접 실습합니다.

1. 허프만 압축이란?

허프만 압축은 자주 등장하는 문자에는 짧은 비트를, 적게 등장하는 문자에는 긴 비트를 부여하는 무손실 압축 알고리즘입니다.

핵심 원리
빈도가 높은 문자 → 짧은 코드
빈도가 낮은 문자 → 긴 코드
시험 포인트
가장 작은 빈도 2개를 계속 합쳐서 이진 트리를 만듭니다.
예: AAAAABBC
A: 5번, B: 2번, C: 1번
A를 짧게, C를 길게 표현하면 전체 비트 수가 줄어든다.

2. 단계별 원리

문자열을 읽고 각 문자의 등장 횟수, 즉 빈도를 계산합니다.
빈도가 가장 작은 두 노드를 고릅니다.
두 노드를 합쳐 새로운 부모 노드를 만듭니다.
노드가 하나 남을 때까지 반복합니다.
왼쪽 간선은 0, 오른쪽 간선은 1로 하여 각 문자의 코드를 구합니다.

6. 연습문제 5개

문제를 모두 풀면 전체 정답과 해설을 볼 수 있습니다.

0 / 5 완료

문제 1

문자열 BANANA에서 A의 등장 횟수는?

문제 2

빈도표 A:7, B:2, C:3, D:8 에서 첫 번째로 합쳐지는 두 문자는?

문제 3

A=0, B=10, C=11일 때 010110을 복원하면?

문제 4

문자열 MISSISSIPPI에서 I의 등장 횟수는?

문제 5

허프만 압축에서 가장 먼저 하는 일은 무엇인가?

전체 정답과 해설 1. BANANA에서 A는 3번 등장합니다. 2. 가장 작은 빈도는 B(2), C(3)이므로 두 문자를 먼저 합칩니다. 3. 010110은 0/A, 10/B, 11/C, 11/C, 0/A 이므로 ABCCA입니다. 4. MISSISSIPPI에서 I는 4번 등장합니다. 5. 허프만 압축은 먼저 문자 빈도표를 만든 뒤, 작은 빈도 2개씩 합칩니다.

7. C언어 구현 핵심 코드

실제 구현에서는 최소 힙을 이용해 가장 작은 빈도 노드 2개를 빠르게 꺼냅니다.

while (heapSize > 1) {
    Node *left = popMin();
    Node *right = popMin();

    Node *parent = makeNode('#', left->freq + right->freq);
    parent->left = left;
    parent->right = right;

    push(parent);
}
